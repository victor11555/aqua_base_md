# Раздел «Отчеты» — База знаний Aqua Delivery

![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро1.png)  
На данный момент для работы доступны следующие отчеты:

**User Acquisition** — отчет по установкам мобильного приложения. Он также содержит информацию об источниках установки приложения. Можно скомпоновать достаточно емкий отчет с помощью сегментирования и настройки группировок с метриками.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/user-acquisition-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро2.png)

**Remarketing** — аналогичный отчет User Acquisition, только с данными ремаркетинговых трекеров (такие трекеры настраиваются при создании — указывается, что это ремаркетинговый трекер).

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/remarketing-report.html

**Аудитории** — отчет по активным пользователям приложения. Количество тех, кто за выбранный период хотя бы один раз запускали приложение.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/audience-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро3.png)

**События** — это отчет по собственным событиям в мобильном приложении. Есть ряд событий, которые настроены на стороне Aqua Delivery. Отчет выводит количественные значения событий. Например, сколько всего раз пользователи авторизовались в приложении.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/events-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро4.png)

**Воронки** — отчет отображает конверсии для заданных в цепочке действий пользователей. Т.е. вы можете строить самостоятельно цепочки действий для изучения поведения пользователей. Например, сколько установили приложение > сколько авторизовалось > сколько сделали первый заказ > сколько отменили заказ.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/funnels-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро5.png)

**Retention-анализ** — отчет показывает, как меняется доля вернувшихся пользователей с течением времени. Другими словами — это то, сколько пользователей останутся пользоваться приложением с течением времени. Чем выше процент пользователей со временем, тем лучше.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/retention-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро6.png)

**Когортный анализ** — отчет позволяет выделить группы пользователей (когорты) и отслеживать показатели совершенных событий за выбранный период времени в когорте. С помощью отчета можно оценить качество источников трафика и качество аудитории, которая пришла через партнеров. Также отчет позволяет анализировать, сколько пользователей совершили событие в выбранной когорте. Таким образом можно рассчитать привлеченных пользователей от определенного партнера.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/cohort-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро7.png)

**Вовлеченность** — отчет «Вовлеченность» отображает информацию о том, сколько пользователь в среднем проводит времени в приложении, как часто запускает приложение, сколько в среднем длится одна сессия, общее количество сессий и общее время в приложении.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/engagement-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро8.png)

**Профили** — отчет отображает распределение пользователей по заданным атрибутам. Данные могут помочь в определении тенденций поведения аудитории по доступным атрибутам.

Например, по атрибуту “операционная система” можно сравнить тенденции двух аудиторий пользователей.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/profile-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро9.png)

**Список профилей** — отчет содержит данные профилей в разрезе каждого отдельного устройства (также содержится информация по дате установки и дате последнего запуска). Отчет является обезличенным (нет конкретного ФИО пользователя).  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро10.png)

**Анализ покупок** — отчет «Анализ покупок» отображает, какие действия совершают пользователи при совершении покупок. Например: открытие страницы, просмотр карточки товара, добавление товара в корзину, оформление заказа.

Важно: система не учитывает отмененные заказы, т.к. является маркетинговым, а не учетным инструментом. Другими словами, будет минимальная погрешность, но для рекламы и анализа аудитории в широком смысле — более чем достаточно.

Подробная инструкция: https://appmetrica.yandex.ru/docs/mobile-reports/concepts/ecommerce-report.html  
![](https://howto.aqua-delivery.com/wp-content/uploads/2024/01/ро11.png)

**Подробнее об отчетах можно прочитать в базе знаний самого сервиса AppMetrica** https://appmetrica.yandex.ru/docs/mobile-reports/concepts/about.html.
